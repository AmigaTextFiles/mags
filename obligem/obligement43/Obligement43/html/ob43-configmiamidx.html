En pratique : MiamiDX - configuration complémentaire par Philippe Bovier



Cet article explique certains points complémentaires d'une configuration logicielle avec MiamiDX lors d'une première connexion. Cet article ne veut pas devenir une référence pour une première connexion mais plutôt espère donner quelques conseils intéressants au sujet de quelques paramètres présents dans une pile TCP/IP.

Introduction sur MiamiInit

En tout premier lieu, il vous faut posséder une installation de MiamiDX sur votre Amiga et ensuite utiliser MiamiInit pour une première connexion sur Internet.

Voir cette page sur comment réaliser sa première connexion sur Internet.

Introduction sur MiamiDX

Après le premier chargement de MiamiDX, il vous faut importer la configuration faite par MiamiInit pour une première utilisation de MiamiDX. Pour cela, utilisez le menu (en version Française sur Aminet) Préférences -> Importer préférences de miamiInit V3.

Tous les menus de MiamiDX sont maintenant remplis avec les préférences de MiamiInit.

Si vous utilisez les menus principaux de MiamiDX, vous retrouvez vos paramètres dans les différents menus :
  • Hardware : vos préférences port série, MTU et modem.
  • Connexion : vos numéros d'appels.
  • Interfaces : vos interfaces de connexion (PPP0 et son alias à remplir si non disponible ex : dialer).
  • Base de données, TCP/IP, évènements, journaux, fenêtres, GUI et socks.

Remarques complémentaires

Voici quelques réponses aux questions que l'on peut se poser la première fois que l'on utilise Miami Deluxe, notamment lorsque l'on se balade dans tous les menus de ce logiciel.

Les options

Si vous explorez tous les menus, vous vous apercevrez visiblement qu'il n'y a pas beaucoup d'options actives après l'importation des préférences de MiamiInit vers MiamiDX, ce qui peut tenter l'utilisateur à lire la documentation en Anglais et de suivre les commentaires faits par l'auteur et ainsi mettre sur ON les options décrites dans la doc.

Je vous conseille de ne pas faire cela et de tester votre connexion telle quelle au départ. Si celle-ci fonctionne bien, vous avez alors gagné jusqu'à 10 secondes sur le temps de connexion par Miami (40 secondes avec options et 30, voire moins, avec configuration telle quelle) par exemple en ajoutant l'option ICMP ou DHCP. Je n'ai d'ailleurs jamais eu d'aussi bons résultats qu'avec une connexion par défaut de MiamInit sans aucun ajout de paramètres TCP/IP par la suite.

Serveurs DNS

Dans le menu Base de données se trouve une liste de paramètres dont "Serveurs DNS" mais qui n'a, jusqu'à maintenant, pas été rempli. Dans le menu Interface -> Réglages TPC/IP, l'option "Donne au serveur un DNS dynamique" se trouve sur vérifier ou ajouter.

A chaque connexion, MiamiDX recherchera vos DNS et perdra du temps de connexion puisque vous les connaissez déjà, dans le fichier journal de MiamInit ou dans les paramètres fournis par courrier par votre provider.

Pour avoir vos paramètres DNS figés, il vous suffit, lors de votre première connexion, d'aller dans le menu Base de données -> serveurs DNS et d'enlever "temp" pour les 2 serveurs DNS de votre fournisseur.

A la fin de votre connexion, il vous suffira de sauvegarder les préférences de MiamiDX puis, à la prochaine connexion, de retourner sur Interfaces -> réglages TCP/IP et de choisir "ignorer" pour l'option "Donne au serveur un DNS dynamique".

Initialisation du modem

MiamiDX a choisi d'office une chaîne d'initialisation pour votre modem. Cette chaîne, qui est plutôt complète (AT&F1&H1&I0&R2&C1&D0\r), n'est pas indispensable et n'a été choisie par MiamiDX que pour une question de sécurité.

La majeure partie de cette chaîne d'initialisation possède des paramètres déjà par défaut dans la ROM du modem. Un modem doit théoriquement pouvoir fonctionner d'office avec sa configuration en ROM. Il y a juste à choisir une configuration matérielle (AT&F1) pour que son modem fonctionne correctement.

Ma chaîne d'initialisation actuelle (AT&F1&K3LO&S32=34\r) possède :
  • K3 : pour une utilisation V42BIS (compression des données automatique mais sans compression des données MNP5. Très utile lors de visite de site Internet et transparence lors de téléchargements de programme archivé sur Aminet par exemple).
  • LO : volume bas du haut parleur.
  • S32=34 : 3 désactive V34, 4 désactive 33600 bauds.

Sortie vers fichiers de Miami

Dans MiamiDX, menu Interface -> Journal -> journal des connexions, il est facilement possible d'obtenir un journal des connexions. Ceci vous permettra, avec des programmes du genre Miamiphone ou Haracz (compteur Internet), de connaître vos communications mensuelles pour ne pas dépasser votre forfait (ex : miami:miami.phone).

Toujours dans cette fenêtre, "Journal PPP" vous donne un journal des paramètres qui sont utilisés lors de l'initialisation de la connexions (ex : miami:miami.PPP).

Vous y verrez le fait que votre provider change votre MTU en 1500 pour votre connexion quelle que soit votre valeur MTU initiale. Ce n'est pas pour cela qu'il faut utiliser votre MTU=1500 dans MiamiDX.

Utilisation de programmes externes

Il est possible graphiquement de vérifier vos transferts Internet avec votre Amiga de 2 façons bien plus pratiques car visuelles :

Par un programme comme MiamiGraph, avec son appel lors de la connexion. Dans le menu Evèmements -> Démarrer.

MiamiGraph PERIOD=50 DIVISIONS=1 BCOL=000000 GCOL=00FF00 DCOL=FF0000 BORDERLESS INTERFACE=ppp0 Par un programme comme GelBesPanel avec son chargement à partir de GUI -> Réglages panneaux de contrôles.
