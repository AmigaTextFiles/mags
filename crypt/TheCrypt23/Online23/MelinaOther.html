Nostalgia

What is Nostalgia ?

Well, in fact, Nostalgia is graphical frontend which goal is to simplify the usage of many consoles emulators on Amiga. "Why another GUI ?", would you say ...

Simple !

Aren't you fed up with all those command lines to type in order to run an emulator ?

Don't you regret all those emulators, so much that sometime you don't know which one you have to use in order to run your game ?

Aren't you fed up with all those GUIs, more or less complete or not always very stable ?

Well, WE are and that's why we created Nostalgia, the only GUI which let's you manage 11 emulators which leads to 13 emulated consoles.

And all this, extremely easy.

This is the end of the Shell or the juggling between emulators, the principle of Nostalgia is very simple, choose your game instead of having headaches asking yourself with the "why" and the "how".

The only thing you have to do is selecting a game and ... play !

>

Features


  • 11 emulators supported => 13 emulated consoles !
  • Simple GUI, nice and intuitive
  • Configuration auto backup
  • Games preview by screen grabs
  • Auto dithering of the GUI, run on any display
  • Easy installation with OS >3.5 support, clean uninstall
  • Localised (French, English and Italian currently)
  • Nostalgia support by the emulators
  • Sortable list, custom rom paths
  • Version checking
  • Skinnable
  • CD games support for PCEngine and Playstation
  • Stable and fast

SkyLights : THE centralized multimedia system !

What is it ?

Well, to be simple, let's say that SkyLight is a GUI grouping many viewers for many file formats. In that way, it can be compared to another of our products : Nostalgia

Anyway, Nostalgia is a "basical" GUI, SkyLights will be much more sophisticated and will be the first product of a new generation of GUIs : The I.G.P.I (Intelligent Graphical Polymorphic Interface).

The SkyLights system (because this is more a complete system than just a GUI) is an open system. It's aim is not to be some sort of monolith reserved to some selected viewers. The interested thing about a polymorphic GUI is the fact that anyone would ba able to develop his player as he wants and then, if he wants that his player to be supported by SkyLights, he just needs to follow some rules (an API in fact) concerning SkyLights.

Anyway, the viewers managed by SkyLights are absolutely not some sort of "plugins" working only under the SkyLights environnement.

They are true players, perfectly independant from SkyLights, the only difference between these players and the others is the fact that their author would have add some features to enable the SkyLights support such as the messenging mechanism which is perfectly transparent for the user.
Features

One unified GUI for thousands of file formats (audio, video, text, etc ...)
Polymorphic interface which is always up to date : This is the viewer which condition the GUI
Skinnable GUI for full customization
Always up to date for all the file formats : specialized engines, generic engines via datatypes, etc etc ...
Optimized for best performances on any hardware thanks to the specialized engines
Open developpement system for easy evolution or integration in third applications
File previewing
Many fonctions : Printing, playlists, file management, slideshows, etc etc ...
Audio CD support



SpiderWeb : THE Global Server




SpiderWeb is a project which aim is to become THE global server solution on Amiga, no more, no less.

But, you would say that there are many tools enabling your Amiga to become a server for this or that service ... That's true but give the name of a tool managing at the same time, an FTP server, IRC, HTTP, files/printers sharing, remote control and an SQL Server ?

Well, that's will do SpiderWeb with, even more, a graphical frontend which will be extremely simple to use in order to manage all these services !

Let's have fun with network administration, transform your Amiga in a multi-services server in a few minutes ;o)

SpiderWeb is a huge project which demand many ressources, we currently have nothing to show about but we are working hard on it hoping the wait won't be too long ...

While waiting, here is a features list :

  • Very easy and customizable GUI.
  • Services management : FTP, IRC, HTTP, files/printers sharing, remote control, groupware mail and SQL server.
  • Profile manager to simplify multi-user control and server moving.
  • Modify/activate/disable services features on the fly, no need to reboot the server !
