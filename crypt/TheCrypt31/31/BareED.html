BareEd Beta Release - Amiga Text Editor

Author: Joerg van de Loo (Joergloo@aol.com)

I have to admit, While surfing through the Aminet I passed over BareED several times before I plucked up enough interest to download it and have a look.

Well after all, Text Editors are one thing the Amiga has plenty of, and supporting a name like "BareED" it really didn't seem to have much to offer.

I assume that every Amiga user will use a text Editor at some time in their lives, I myself use three text editors on a regular basis


Edword Pro:- My preferred editor is used for almost all formatting work.

GoldED:- Used mainly for its excellent html Plug in

Article Editor :- Came batched with Propage, Has the excellent feature of "Save highlight region" Excellent for chopping up large files or extracting certain areas of text.


It will be noted that each editor is used for it certain ability to complete a function, So what could BareED have to offer that these excellent editors don't.

Well first of all don't get confused about the choice of name for this program, A brief look at the Programmer's notes (Detailed below) Shows the BareED is actually quite a powerful tool, And hopefully later versions will further improve the program into a complete, usable Text Editor.
Authors Notes:

Simple text editor which supports proportional fonts.

Simple text editor which can support proportional fonts and graphic cards. Present in the beta stage. Still missing such nice things like folding, multiple projects and much more, offers however already in this version relief for proportional fonts as well as variable colours.

Does not use the Amiga console driver but the graphics primitives.
Requires OS 3 and a 68020 CPU.


- CLI and WB interface
- detachment of the CLI
- no fixed, defaulted colours
- cooperates with all Amiga fonts, exception: colour fonts
- will not modify the protection bits of a file careless
- icon saves with alternative icon image support
- deals with simple- and smart-refresh windows
- larding note (clipboard)
- text formatting for normal and AmigaGuide-documents
- short cuts
- bookmarks
- prepared to be multilingual
- quite quick search function
- button interface
- button interface offers to start ARexx scripts
- export of archive to IBM (compatible) / MAC platform
- simple revision control for authors
- DAC interface
- backtracking Undo-functions (now really useable)
- graphical page dumper
- although very stable on my system some faults left
- source code not enclosed
- enclosed sources for start-up-code (C), memory copy routine (Asm) and for page dumper (C) including header files
- comes along with a few ARexx scripts




Now the part that really caught my eye, Was the ability of BareED to use Amiga Fonts. I often use a text editor to write/Print short notes (Rather than load up Wordworth or any other wordProcessor) However most editors only allow you to use a basic font, BareED suddenly started to look very interesting.





As you can see from the above Graphic, With BareED you can select the font you want, and also adjust the Font Colour, and background colour.





Tests on printing using BareED and Workbench fonts, have shown that the Fonts selected will be used in printing, However at the moment letters (etc) will only print in black. However this is of course only a Beta version of, What appears to be a nice Text Edtor, and I just hope that suffient interest will be shown in it, to incourage the Programmer to continue with the work.
