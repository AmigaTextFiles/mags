Installation

 
Na, einfach irgendwo hinkopieren, was sonst? Und MPEGA sollte man irgendwo auf der Platte installiert haben. Das Programm fragt beim ersten Start gleich die genaue Pfadangabe ab und speichert diese, sowie das Default-Directory. MPEGA ist ein Shell-Utility zum Abspielen von MP3s:


MPEGA 3.3 (29.8.97) [68040/60] (C)1995-1997 Stephane TAVENARD
Email: tavenard@xiii.univ-angers.fr
Giftware MPEG Audio decoder for Layers I,II and III

Usage: mpega [<options>] <MPEG Audio file> [<out file>]
Options:
-d <freq_div> frequency division: 1, 2 or 4 (default=1)
-q <quality> output quality: 0 (worst), 1 or 2 (best) (default=2)
-h display header only
-n no frame counter
-m mono output (left channel)
-p <pri> process priority (-30..30)
-I[<id>] AHI output, <id> = Audio mode ID in hex (ex: -I20004)
-s silence (no audio output)
-f 0|1 audio filter off/on (default = auto)
-x <freq> audio mixing frequency (default = none)
-t <ms> audio buffer time in ms (default = 1 sec)
-v <vol> audio volume (0..64 or 65..256 for boost)
-w wait for audio buffer to be full before starting audio
-a audio buffer fill level in secs (use with -T option)
-S <ms> seek to time postion in stream (unit is ms)
-A write AIFF header to output file
-W write WAVE header to output file
-F <freq_max> max decoding frequency (auto freq_div)
-T display time counter instead of frame counter
-b <k_size> set the input file buffer size in KBytes (default = 16KB)
-D display decoding parameters
-l input file is a play list (text file)
-o <out file> output decoded file
-c <cfg file> configuration file (default = MPEGA.CFG)
-C display current configuration
-u display time used
-r don't use Async I/O for input file (no lock)
-N display filename
-R random play
-i display TAG infos

Enjoy fast MPEG :-)

 
Zusätzliche Informationen.
 
 

Features

 
Mit MpegA kann man aus MPEG-Dateien AIFF- oder WAV-Samples erzeugen, und dies geht sogar recht flott vonstatten, auf meiner WarpEngine in Stereo, höchste Qualität, 44.1kHz dauerte der Dekodierungsprozeß während eine andere MPEG-Datei vom selben Programm abgespielt wurde nicht einmal eine Minute. Das File war ca. 5 Minuten lang und eine Standard MPEG II Layer 3 Datei, allerdings auch nur knapp ein MB groß. Das erzeugt AIFF hatte danach die zehnfache Größe, aber wozu hab ich 78 von 80MB FAST-RAM frei?

 

Fazit

 
Als Player ist die MpegA-GUI eher dürftig, aber ausreichend, die Abspielliste wird gespeichert und gleich nach dem Start des Programms darf man im Listview nach den Files wühlen, mittels PatternMatching werden auch nur die wahren MPEG-Dateien angezeigt. Zum Dekodieren wäre noch zu sagen, daß man sich zwar den Spielbeginn aussuchen darf, aber das File gnadenlos zu Ende gesampelt wird, man kann nicht 'auf Stop drücken'. Naja, wer MPEGA hat, und sich mit einer GUI leichter tut, dem sei das Programm gegönnt.
