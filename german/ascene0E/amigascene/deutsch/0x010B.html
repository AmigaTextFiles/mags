GroovyPlayer - Get into the Groove, Oida!

 
Täter : Miklós Árpád (arpikata@elender.hu)
Aktennummer : 1.12
Quellen : aminet:disk/cdrom/GroovyPlayer.lha
Prämie : Freeware
Anforderungen : OS 3.x, CD-ROM, 200K Free Memory
 
 
Lange Zeit war ich auf dem Amiga auf der Suche nach einem CD-Player mit einer schönen, einem richtigen CD-Player nachempfundenen GUI. Keines der am Aminet befindlichen Programme konnte mich jedoch richtig zufriedenstellen. Dies änderte sich jedoch schlagartig, als ich mit dem GroovyPlayer Bekanntschaft machte. Dieser hatte eine GUI, so wie ich es mir schon immer gewünscht hatte. Installation problemlos, wie gewohnt. Der Eintrag für das Standardprogramm zum Apspielen von Audiotracks wird sogleich umgestellt (vorher hatte ich den OptyCDPlayer benutzt), und nach einem Doppelklick auf das Icon erscheint sofort die komfortable Oberfläche:

 
Ein Klick auf "List" und ich kann die Namen der Tracks und Titel eingeben, wie man es von anderen Playern gewohnt ist. Eine kleine Database mit ein paar IDs ist auch dabei, aber bei meinem Musikgeschmack passiert es mir eher selten, daß eine meiner CDs erkannt werden, und von den gängigeren Gruppen hab ich wiederum nur Bootlegs. Egal, ich schnappe die nextbeste CD und geh' schon mal die Tracks eingeben. Auch wenn es wahrscheinlich die letzte CD ist, bei der ich dies tun werde (wer braucht schon Trackangaben, die stehen doch eh auf der Hülle ;-).

Das Programm bietet noch einige interessante Features, wie zum Beispiel einen TapeMode, mit dem man sich eine Trackzusammenstellung generieren lassen kann, sodaß die Tracks ohne Unterbechung auf beiden Seiten der Kassette Platz finden. Einziges Manko, das ich bei mir festgestellt habe, ist, daß der Stop-Button nicht funktioniert (mein CD-Laufwerk zeigt sich einfach unbeeindruckt und spielt ohne Unterbrechung fort). In dem Fall helfe ich mir eben mit dem Eject-Button. Volume, Balance und sogar eine Mute-Funktion wurden dem Programm spendiert. Mit "Clear" oder "Stop" steigt man aus dem Programm-Modus aus, kann ihn aber mit Hilfe von "Recall" wieder aktivieren. Praktisch. Die restlichen Bedienelemente sind die, die man auch auf jedem Standard CD Player in natura findet, deren Beschreibung erspare ich mir.

 
Weil ich so faul bin, die Features aus dem GroovyPlayer.guide:

Supports standard SCSI-2 and ATAPI drives
All standard CD player functions including play/pause, skip, search, stop, eject and random track access
Nice front-end that adapts to your screen depth and palette (a la Empy...)
Track list including name of author, CD title and track titles, compatible with MCDPlayer (so you can use your saved list files)
Double-clicking on a track name in the List window will start playing that track
'Smart List' switch enables the user to program the player from the list window, which appears and disappears automagically as discs are inserted/ejected
List window shows the number and length of the track selected/being played
Support for multimedia CD's (compact discs with data track(s) and at least one audio track)
Adjustable volume (256 levels) and balance (left/right 0% to 100%)
Logarithmic volume option (for smoother and faster control)
Mute function (3 selectable gain levels)
Random shuffle
Repeat mode (repeat all tracks, programmed tracks or tape selection)
Intro mode (listen to the first x seconds of each track)
Fade out (immediately or at the end of the intro)
Program mode (max. 100 tracks; recall button for retrieving finished programs)
Tape mode (selectable/user definable tape length (like C-60 or C-90), layout tracks to side A/B of tape)
Adjustable preferences with various settings
Informative LCD-like display including current track and index, track numbers, titles, several time counters and a status indicator LED (play/pause/ffwd/rev)
Programmed entirely in MC68020 assembly (fast!)
 
In den Voreinstellungen kann man allerhand definieren, so zum Beispiel die Zeit, die verstrichen sein muß, damit der Player bei einem "Skip Back" nicht zum Anfang des aktuellen Tracks, sondern zum vorherigen Track springt. Der Player kann als Icon oder ins Tool-Menu ikonifiziert werden, benutzt kein MUI, man darf den PublicScreen wählen, das Verzeichnis für die CD-IDs bestimmen, Trackverzögerungen einstellen, sogar die "Füße", auf denen der CD-Player steht, lassen sich abschalten.

 
Fazit: Programm ist Freeware, braucht kein MUI, kommt mit wenig Ressourcen aus, hat ein ganz nettes Interface und kann - was ja das Wichtigste ist - CDs abspielen, mehr wollen wir ja garnicht. Bestens.
