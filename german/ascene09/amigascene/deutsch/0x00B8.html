Aminet Uploads - Wie lade ich was auf's Aminet?

 
Um den Sauhaufen, der auf den meisten FTP-Sites vorherrscht, zu unterbinden, gibt es für das Aminet einige Regeln, die vom Uploader beachtet werden müssen. Dies ermöglicht den zahlreichen Benutzern dieses Netzes das komfortable Durchsehen und Herunterladen der Archive.

 
In diesem Artikel werden wir uns an einem Beispiel ansehen, wie man in der Praxis so einen Upload durchführt. Ausführlichere Informationen über die Regeln des Uploads finden sich übrigens in der Datei UploadToAminet.txt welche selbst am Aminet unter aminet:docs/help/UploadToAminet.txt zu finden ist.

In unserem Beispiel wollen wir ein Softwarepaket Namens mCWEB, das aus einem Verzeichnis mit zahlreichen Dateien besteht, aufs Aminet spielen. Dabei handelt es sich übrigens um ein Literate Programming System, einem Nachfolger des bekannten CWEB, welches dieses um Funktionalitäten erweitert, die für die Teamarbeit notwendig sind.

 
Für alles, was aufs Aminet gespielt wird, muß eine Readme-Datei angelegt werden, welche dann ebenfalls hinaufgespielt werden muß. Der Name der Readme-Datei muß gleich wie der des Archives sein, jedoch als Dateierweiterung ein .readme besitzen. Wenn unser Archiv also mCWEB.lha heißt, dann müssen wir zusätzlich eine Datei mCWEB.readme aufs Aminet spielen. In dieser Datei stehen die Informationen über unser Archiv, die sich der Benutzer ansehen kann, ohne, daß er das ganze Archiv herunterladet.

Der Aufbau dieser Readme-Datei ist jedoch nicht vollkommen frei, sondern muß gewisse Felder enthalten. Diese Felder bedeuten folgendes:


Short:
Dies ist das einzige zwingend vorgeschriebene Feld, und es muß in der ersten Zeile stehen. Hier kann eine kurze (max. 40 Buchstaben) Beschreibung des Uploads eingegeben werden, der später in INDEX und RECENT angezeigt wird, so daß jeder auf einfachste Weise Informationen über den Upload erhält. Hier sollte der Dateiname nicht wiederholt werden, aber wenn mehrere Versionen des Archivs existieren, kann hier die Versionsnummer angegeben werden. Es soll erklären, was das Programm tut.

Uploader:
Hier kann die E-Mail-Adresse des Uploaders angeben werden, sodaß dieser bei Problemen kontaktiert werden kann.

Author:
In diesem Feld kann angegeben werden, wer die upgeloadete Software erstellt hat.

Type:
An dieser Stelle kann das Verzeichnis angegeben werden, in das das File hineinkommen soll. Man prüfe das File TREE nach möglichen Unterverzeichnissen. Wer ein neues Verzeichnis will, liest 'info/start/newdir.txt'.

Kurz:
Dieses Feld ist die deutsche Version von 'Short:'.

Die übrigen Felder sind vollkommen optional:

Replaces:
Hiermit können Files angeben werden, die von Upload ersetzt werden sollen. Es muß der vollen Pfad angegeben werden, z.B. biz/patch/PageStreamPatch*. Das Feld ist nicht nötig, wenn ein früheres File mit demselben Namen überschrieben wird.

Requires:
Andere Archive, die der Upload zum Funktionieren benötigt, können hier mit vollem Pfad genannt werden. Auch OS-, Speicher- und Chipset-Bedürfnisse können hier angegeben werden.

Version:
Die Versionsnummer des Uploads. In den Dateinamen sollte nach Möglichkeit keine Versionsnummer enthalten sein.

Distribution:
Hier kann angegeben werden, wo der Upload verbreitet werden darf. Wenn das Schlüsselwort "NoCD" genannt wird, wird der Upload auf keiner Aminet-CD veröffentlicht. Wenn "Aminet" spezifiziert wird, erhält nur das Aminet die Erlaubnis zur Verbreitung.

 
 
Nach einer Leerzeile kann eine längere Beschreibung angefügt werden, z.B. das Liesmich-File aus dem Archiv. Da nicht alle Benutzer das .readme-File downloaden, sollten alle Infos daraus auch im Archiv vorhanden sein. Unser mCWEB.readme sieht folgendermaßen aus:


Short:   Literate Programming System for teams
Uploader: oelli@eunet.at (Thomas Öllinger)
Author:   Markus Öllinger
Type:   dev/c
Kurz:   Literarisches Programmiersys. für Teams
Version:  1.1

This archive contains binary, source and documentation of mCWEB -- a descendant of the CWEB system of structured documentation by Donald E. Knuth and Silvio Levy -- that adds some features that are indispensable when working in a team. mCWEB regards a project as a book consisting of several chapter files. By means of import and export commands, it automatically manages all relationships between the chapters of a book and to other books.

Interface Documentation is now part of the mCWEB file and can be extracted into a second TeX file. This allows to define well known interfaces between the individual parts of a project that will be implemented by different persons.

mCWEB is a real extension to CWEB which means that all of the original code is part of mCWEB. mCWEB's command set is a superset of CWEB's commands. mCWEB is backwards compatible so that you can also translate old-style CWEB programs.

Copyright (C) 1996-1998 Markus Öllinger

 
 
Bevor wir unser Programm uploaden, müssen wir einmal unser ganzes Programmverzeichnis in ein LHA-Archiv packen, denn auf das Aminet dürfen nur mit lha gepackte Archive gespielt werden. Außer, es handelt sich um einzelne Grafiken oder Animationen, welche auch als .jpg- und .mpg-Dateien erlaubt sind. Dazu stellen wir uns in das Verzeichnis, in dem sich unser Programmverzeichnis befindet und erzeugen ein LHA-Archiv. Bei unserem Beispiel befinden sich die Dateien im Verzeichnis Work:Proj/mCWEB.


cd Work:Proj
lha -r a mCWEB.lha mCWEB

 
Dadurch wird erreicht, daß alles im Verzeichnis mCWEB und darunter in das LHA-Archiv mCWEB.lha gepackt wird. Außerdem wird später beim Entpacken wieder das Verzeichnis mCWEB angelegt, wodurch alle dazugehörigen Dateien dann schön durch dieses zusammengehalten werden. Denn es gibt nichts aufreibenderes als Archive, die einem beim Entpacken mit hunderttausend Einzeldateien die RAM-Disk versauen. Besonders komfortabel ist es, wenn beim Entpacken gleich ein Icon für das mCWEB Verzeichnis mitangelegt wird. Daher fügen wir dieses noch zusätzlich zu unserem Archiv dazu:


lha a mCWEB.lha mCWEB.info

 
Beim Namen des Archives muß übrigens beachtet werden, daß dieser nicht länger als 18 Zeichen inklusive des Archiver-Suffixes (.lha) ist. Groß-/Kleinschreibung ist erlaubt, aber keine Sonderzeichen (wie Umlaute). Also nur a-z, A-Z, 0-9 sowie Unter- und Bindestrich. Außerdem sollten die ersten 8 Zeichen des Namens eindeutig sein, das heißt mit keinem der bereits im gleichen Verzeichnis am Aminet befindlichen Dateinamen übereinstimmen.

 
So, wir haben jetzt unsere zwei Dateien mCWEB.lha und mCWEB.readme, die aufs Aminet sollen. Das Aminet besteht aus zahlreichen FTP-Sites, die ihre Dateien untereinander austauschen. Die zentrale Verwaltung ist wuarchive.wustl.edu [128.252.135.4]. Nach Möglichkeit sollten die Dateien dort upgeloadet werden. Die meißten anderen Sites schicken die Dateien nämlich auch an diese Site und löschen sie zunächst lokal. Später werden sie dann wieder zurückgespiegelt. Auf keinen Fall sollte man die Dateien auf mehreren Sites uploaden. Auf den Aminet-Sites gibt es jeweils zwei Upload-Verzeichnisse:


pub/aminet/new für öffentliche Uploads
pub/aminet/priv  für private Uploads (nur in der jeweiligen Site)

 
Wir betrachten hier nur öffentliche Uploads. Was es mit privaten auf sich hat, kann in der Datei UploadToAminet.txt nachgelesen werden.
