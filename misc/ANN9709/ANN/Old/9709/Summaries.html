AROS advantages (26-Sep-97 )

    Lars Bischoff reminds that AROS has got many advantages over other OSes:
    • it is platform-independent
    • it is binary-compatible on 68k computers,
    • it is source-compatible on other computers
    • is written to support new features, like memory protection and retargability; although the current version only has the same features as OS 3.1
    • it is written by Amiga users, which means it is possible to implement what users want.
    • And, most importantly, AROS is freely available and does not depend on any company. That means no copyright and licensing problems and no company that can file for bankruptcy.

PowerUp history; according to Phase5 (09-Sep-97 )

    The choice to go PowerPC was taken early october '95 when Motorola began questioning volume production of the 68060. An alpha prototype was shown on the Computer95 show; estimated time of arrival (ETA) was second part of 1996. They then lost 4 months talking about possible cooperations wih AT and then had to face an etire different situation with the bankruptcy of Escom/AT. Another 4 months passed, there were talks with possible Amiga buyers, but these were useless. Since there didn't seem to be a solution to the AT (and AmigaOS) problem, they decided to adopt the current dual-processor concept and began developing prototypes. These were delivered to developers shortly before the Computer96, where they also showed the first small demo application. ETA then was the first quarter 1997. Delays since then were caused by restructuring in the development team and the transforming of design to "VHDL-synthesis" [?]. The last 5 months of delays are Phase5's fault, although this time was also used to further improve the design of the PowerUp boards and software.

PowerUp 68k/PPC concept explained (05-Sep-97 )

    Craig H. Ganoe quotes one of Phase5's web pages that explains how the PowerUp will run.
    While the original AmigaOS 3.x continues to run on the 68k CPU - as well as all existing applications do, what provides 100% compatibility - the PowerPC CPU is supported by a comprehensive PPC Library developed by phase 5, which provides all functionality to integrate the PowerPC into the Amiga multitasking environment. Software developers can easily optimze their programs step by step by porting single tasks to PowerPC code, which then can run parallel with 68k tasks.
    Craig reads this as AmigaOS 3.1 running on the 68k and if a programmer wants to run PPC code, the programmer creates a subtask on the PPC using the PPC.library. Looks to me all the AmigaOS is the same old 68k code (which makes sense to me since this is all Copyrighted) and the programmer can write their own PPC subtasks to do stuff for them.

Arguments against the Phase5 concept (02-Sep-97 )

    If AI made an AmigaOS 4.0 CHRP, all people who have bought PowerUp cards would start whining. AI would have to decide whether to support an additional (totally different) hardware architecture or not. The PPC chip is the same, but the hardware interface to the system is totally different, this is the hard part [..] of porting the OS, because you have to write totally different code for every architecture you support. Phase5 did the Amiga a service by developing 040/060 accelerators and advanced graphics and SCSI solutions, but these new PPC boards of theirs are their last shot at making money off the Amiga before we see a shiny new PPC Amiga with PCI and all that good stuff, says Craig H. Ganoe. What concerns him are people blindly preordering Phase5 PPC cards or anxiously awaiting their release, not even understanding how they work. People really need to wait and understand and see the big picture.
