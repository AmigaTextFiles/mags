Good/bad things about WarpOS (04-Oct-97 )

    Steven Matty posts a good/bad things list about WarpOS. The only "good thing" is that WarpOS is (allegedly) faster. The fact that programmers need to use H&P's commercial C compiler (and not GCC or SAS/C) to develop for it is definately a bad thing; besides that it is not compatible to the de-facto PowerUP kernel; it doesn't run programs compiled to use Phase5's library in ELF format; one needs to reboot to run a PowerUp or WarpUp program and, finally, WarpUp is a kludge based on reverse-engineering of the PowerUp hardware. Steve says he thinks this is a disguised attempt to force Amiga developers to use the StormC compiler as "the standard".
