 ¿ Qué es WebPlug ?
-==================-

WebPlug es un editor de páginas HTML para el Amiga.

El formato de descripción de páginas HTML es el que se utiliza para crear y
mantener páginas Web en Internet.

Se  puede  (muy  remotamente)  equiparar  un  fichero HTML a un AmigaGuide.
Simplemente  son  ficheros  de  texto  que contienen códigos de control que
permiten  al  navegador  (el  programa  que interpreta estos ficheros HTML)
presentar el documento en pantalla.

Evidentemente,  HTML  da muchísimo más control sobre el aspecto final de la
página  que  AmigaGuide.   Es  posible  definir imágenes, fondos, enlaces a
otras  páginas,  crear  formularios  (que  permiten  introducir datos en la
página directamente), incluir sonidos, animaciones, programas, etc...

Esto  hace que HTML esté empezando a usarse no sólo para crear páginas para
Internet,  sinó  para presentar documentos por ordenador de forma mucho más
agradable (ahí tenéis el hecho de que la documentación del Photogenics v2.0
venga  en  formato HTML, con el navegador AWeb para visualizarla, en vez de
en soporte de papel o en AmigaGuide).

Es  relativamente  sencillo  crear  ficheros AmigaGuide 'a mano', es decir,
cojer un editor de textos y ponerse a teclear el fichero directamente.  Con
HTML  la  cosa cambia, ya que es mucho más completo, y el número de códigos
que  tiene  es muy grande (y aumenta dia a dia).  Por ello, decidí crear un
programa  que  ayudara  al  usuario a confeccionar sus propias páginas Web.
WebPlug es el resultado.


 Características del programa
-===============-===-========-

Bien,  aquí tendremos que tocar algunos aspectos técnicos referentes a HTML
(lo siento :-).

-WebPlug usa MUI (esto va a cambiar, muy posiblemente).  Requiere MUI 3.2+.

-Está completamente 'localizado'.  Viene con catálogos en español, catalán,
sueco, alemán, francés...  (más en proceso).

-Incorpora  su propio editor de textos, que se basa en el Textfield.gadget.
Este gadget viene con el paquete de distribución del programa.

-Soporta   la   mayoría  de  comandos  HTML  3.2,  esto  incluye  centrados
(
), alineaciones, texto flotante, fondos, etc...

De  momento  el  programa  no  soporta  paneles  (frames),  pero lo hará en
versiones  posteriores.   De  todas formas, actualmente ningún navegador de
Amiga soporta los paneles completamente.

La  v1.2 en la que estoy trabajando incluye una lista de tags configurable,
con  lo  que  es possible crear y utilizar tags HTML que el programa aún no
soporta directamente.

Permite  crear  tablas,  y  es  capaz  de generar automáticamente tablas de
cualquier dimesión (hasta 100×100, vaya).

-Es  capaz de traducir los carácteres especiales de un bloque de texto a su
equivalente  en  HTML (en HTML, 'á' es '&acute;', para evitar problemas con
los  distintos  juegos de carácteres; un documento HTML se puede visualizar
en  cualquier  ordenador  directamente, sin tener que realizar conversiones
entre los juegos de carácteres como hace el CrossDOS, por ejemplo).

-WebPlug  utiliza  unas  librerias  externas, llamadas BrowserLinks, que le
permiten  comunicarse  con los navegadores.  Esto permite al usuario enviar
el  documento  actual  al  navegador  para visualizar los resultados con un
simple click del ratón.

Para  que  un  navegador  sea  utilizable con WebPlug, sólo debe incorporar
algunos  comandos ARexx.  De momento, WebPlug soporta el IBrowse, el AWeb y
el AMosaic.

Soportar  nuevos  navegadores  es  simplemente  cuestión  de  crear  nuevas
librerías para estos.

-Es muy simple de utilizar, comparado con otros programas del mismo estilo.

-Es  muy  configurable.   A  parte  de  ser  MUI, lo que permite cambiar el
aspecto externo del programa, WebPlug incorpora una ventana de preferencias
en  la  que  se puede especificar el tipo de letra del editor, qué ventanas
quieres  abiertas  por  defecto,  el  camino  por defecto de los documentos
HTML...

 Futuro
-======-


Como  cualquier  programa,  WebPlug  no está acabado (ni cerca de estarlo).
Entre  las consideraciones más inmediatas que estoy haciendo está la de que
la v1.2 sea la última que utilice el MUI.

El  porqué  es  algo largo de explicar y no viene al caso en este artículo.
Si quieres más información, consulta en mi página.

opinan  de  portar  WebPlug  a  ClassAct  (el  sistema que usa el AWeb o el
Voodoo,  por ejemplo).  De todas formas, agradecería cualquier opinión.  Me
podéis encontrar en la dirección e-mail:
