UAE speeds (30-Jul-97 )

    On a P200 [..], UAE feels like a slow A3000. To calculate the speed of an UAE-emulated Amiga, you can use this calculation:
    a = average number of x86 instructions per 68K instruction: 40
    b = average number of x86 instructions per clock cycle: 1.2
    c = average clock cycles per 68000 instruction: 3
    Assuming a 100 MHz host CPU, the virtual 68000 runs at a speed of
    100 MHz/a*b*c = 9 MHz
    Apparently, a PPro-200 is exactly the type of CPU UAE starts making sense on
     · Posted by Mathias Ortmann on Mon, 28 Jul 1997 20:43:09 GMT in comp.sys.amiga.advocacy [L].

MUI and buggy programs (22-Jul-97 )

He recalls the old days where everything needed to be programmed without any flaw or the program would crash and that nowadays programming MUI is so easy that people don't even think about testing their programs before releasing them. Index information, part 2 (17-Jul-97 )

    Once again, Mick Tinker clears up what the Access is: a machine for corporate applications. Prices at the moment are $633 for a 1 off cased unit and $412 for a motherboard unit. The motherboard includes a '020, 2 Mb CHIP, 2 Mb FAST, 256kb FLASH ROM, 1kb EEPROM, Real-Time Clock, Sound sampler, IDE, RGB & Composite outputs on rear, Mouse, keyboard & printer ports on rear, Headphone, volume, microphone and reset on font, Floppy drive, Mounting brackets, front & rear cover, Most of the usual A1200 stuff. 8 Mb of FAST RAM wouldn't be a major problem, besides that they are working on (optional) MPEG1 and Ethernet support. Their target for the next twelve months is 10,000 units.
    The Connect is in design and the final ship date is not [yet] certain. It is for retail sale and basically is a low cost A4000. The idea is to start a PC style OEM computer assembly market.
    There are not much informations about the InsideOut, as they don't want to hold people back from spending money and killing the market, they want people to buy what is available otherwise there will be no Amiga market left to sell to. Additionally, they don't want to influence other companies' behaviour (and don't want that they 'borrow' ideas). You'll have to wait and see!

Changing CygnusED... (14-Jul-97 )

    Olaf Barthel says enhancing CygnusED is pretty hard, because it was built upon hard design principles that are nearly impossible to change. Basically, CygnusED is just one blazingly fast text display engine with editing features glued to it. It will probably never offer features like syntax colouring or plug-in API, as that would be too hard to implement into the existing code. Apparently, most of the 50k of suggestions he received already circle around turning CygnusEd into GoldEd, which is obviously not possible.

AmigaBasic on modern Amigas (12-Jul-97 )

    Denis Zwornaz repeats what should be done to run AmigaBasic on modern Amigas. Turn FastMem off, turn DisplayBeep in the Sound prefs off, turn the CPU data-cache off, start AmigaBasic, enter clear 50000 (or a higher value), reactivate fast memory and run your programs. Of course it is more comfortable to use a modern program that is compatible to Amiga-Basic, like ACE or MaxonBASIC (aka. HiSoft BASIC).
