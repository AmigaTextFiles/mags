3. Software test: Script editing tool 'Skriptmanager' (by Uwe Pannecke)

»Everything Scripts - or what?

1. What's it about?

With my fishing rod, I held into the never ending Amiga software source AMINET, I catched a very special fish, the last time. This fish gives help to any AmigaGuide writers, ARexx programmers and Script creators.

This program, named "Skriptmanager" in its revision 1.5 [1] was tested extensively.


Heiko Strohmeier [2] praises the features of his Skriptmanager in the ReadMe text of the Lha-package: 
- Mailing, writing or editing, executing, checking of scripts 
(ARexx/DOS)
  • integrated ARexx-dictionary using an interactive database 
  • an ARexx port allows using the personal favourite editor 
  • Skriptmanager controlled creation of an AmigaGuide using an userfriendly GUI without acrobatics in AmigaGuide control codes
  • Guide2HTML converter function
If the Skriptmanager can keep its promisses, it had to prove on an A3000 with 36 MB RAM and a 040/40 processor under AmigaOS 3.1 and CyberGFX 3.x.

2. System requirements

The Skriptmanager is quite lean. It just requires an Amiga with OS 2.x and 1 MB of Fast-RAM.

3. Installation

If you got the LhA-package and unarchived it, the installation itself is no problem. The Skriptmanager drawer is just copied to the desired hard disk partition and the Install-libs68xxx-icon, which can be found in the open drawer has to be doubleclicked. The install script copies the needed CanDo and PowerPacker library versions to Libs:, depending on the CPU.

But it isn't checked, if there might be a (newer) library, so it is overwritten, if present.

4. Configuration

If SkriptManager found its place on your hard disk, it has to be configured before its first use. The program author offers an own program for this, EditPrefs. After doubleclicking any data and paths required by SkriptManager can be set.

With the opened EditPrefs window the origin of the program can be recognized.

It was programmed using CanDo, a BASIC dialect, which sometimes goes its own ways for creating a graphical user interface. If you want to change the settings, just click on the windows to get a requester, that allows the selection of the desired parameters.

When selecting a font you should avoid using proportional fonts, so the GUI doesn't get "dizzy". The selected font is used in any program window, even in EditPrefs.

After the configuration EditPrefs likes to know, if the settings have to be stored permanently or temporarily and then starts SkriptManager, or more correct its editor, named Power-Edit.

5. Using the program

This editor is the control center for any action.

In the simpliest way you can use it as simple text editor.

But then you have to get use to its headstrong useage. And there are much more powerful freeware editors. But SkriptManager offers more interesting things, which cannot be found with other editors like in SkriptManager:

5.1 Creating Guides

BEFORE you can create a guide you should read the included manual. The Try-And-Error-method will just lead to frustration. Commondably SkriptManager offers an online-help, a press on the HELP-button is enough to show the AmigaGuide manual on the screen. Unfortunately the online-help isn't started synchronous and as long as the help guide is hown SkriptManager is blocked. so unfortunately a parallel use of help and program is not possible.

Because the program describes the necessary steps in some parts, I want to describe how to create a guide.


Imagine, you want to write a guide, that has to contain some links to the remarks - a preface,
- the main section itself, which includes the functions of the fictive program, you developped,
- a shareware notice,
- the program author with account informations etc. and
- a history of the fictive program.

The desired texts of the guide sections you want to join can be written as unique text files and saved giving them good names.

Now, it's time to use the SkriptManager.

If you had to set each of the control codes manually, when creating a guide, the Guidebuilder of SkriptManager does this job, now.

After doubleclicking gadget "GB" or selecting the menupoint "MakeGuide/Guidekonzept/Neues Projekt" in the editor and entering a name for the guide to be created the GuideManager opens.

Now you have to enter the desired link using "Bezeichnungen" a help text, which will appear near the link, using "Linkkomentar" and then to load the previously created text section, which follows the link using "Einzbindender Text" and a click on the "Datei" gadget.

The right link route is done by GuideBuilder.

In addition to that you can set a main page or load a unique page for manipulation into the editor.

Using the help of the gadget "Einfü" any necessary settings can be made for further links.

If this is done a click on the gadget "LinkMan." starts the LinkManager.

This finally defines on which pages of the guide the links will appear.

Wir gehen in unserem Beispiel davon aus, dass alle fuenf Links auf der Hauptseite, die mit dem Oeffnen des Guides als erstes angezeigt wird, praesentiert werden sollen. Hierzu ist im Feld rechts neben "Main" eine 5 einzutragen. Dies teilt dem LinkManager mit, dass fuenf Links auf der Hauptseite abgebildet werden. Sind mehr als fuenf Links vorhanden, schnappt sich der LinkManager die ersten Fuenf aus der Linkliste.

Innerhalb der Linkliste kann bei Bedarf ein Link beliebig mit "L UP" und "L Down" verschoben werden.

Ein Klick auf das "Erzeuge"-Gagdet veranlasst nun die Erstellung des Guides. Gleichzeitig wird das erstellte Guide in den Skriptmanager geladen und steht dort fuer Veraenderungen bereit.

Natuerlich waere es schoen, einen ersten optischen Eindruck von unserem Guide zu bekommen.

Ein Klick auf "Test"- und schon prangt das Guide auf dem Bildschirm, vom Guidebuilder auf der Eingangsseite zusaetzlich mit Eurem Namen samt Anschrift geschmueckt.

Um zusaetzliche Querverweise in den Guide-Text einzubauen, erlaubt der Linkmanager die Auswahl sogenannter UnterNodes, gekennzeichnet in der Linkliste durch "UN:". Ein Klick auf einen Link gibt einen kurzen Hilfstext aus und ermoeglicht in den beiden Eingabefeldern oberhalb von "Main:" die Festlegung der UnterNodes.

Um beispielsweise auf der Vorwortseite Links zu allen weiteren Guide-Teilen zu plazieren, gebt Ihr im ersten Feld eine 2 ein und im zweiten Feld eine 5. Dies bedeutet, dass auf der angewaehlten Linkseite (in diesem Fall Vorwort) die Links 2-5 erscheinen.

Link 1 (also der Link zum Vorwort selbst) soll hier natuerlich nicht abgebildet werden, deshalb die Eintraege 2 und 5.

Soll auf der Sharewareseite ein Verweis zum Programmautor erfolgen, so waehlt man den passenden Link (hier Shareware) mit einem Klick an und gibt in die beiden Felder oberhalb von "Main:" jeweils eine 4 ein. So wird festgelegt, dass auf der Sharewareseite nur der Link Nummer 4 (also der Link zum Autor) erscheint.

Nach der entsprechenden Wahl der Linkverknuepfungen braucht nur noch "Erzeuge" gewaehlt werden, und schon ist das modifizierte Guide fertig. "Test" sorgt auch hier wieder fuer ein Preview.

Soll der Schriftstil von Normal auf Fett, Kursiv oder Unterstrichen geaendert werden, oder moechtet Ihr die Schrift bzw. den Schrifthintergrund farblich aendern, stehen auch dafuer Menue-Punkte zur Auswahl.

Darueberhinaus koennen bereits fertige Guides eingelesen und mit anderen Guides verknuept oder eigenen Vorstellungen angepasst werden.

Interessant ist die AppWindow-Funktion im GuideBuilder. Fuer jede Datei, Schublade, jedes Festplattenverzeichnis oder gar komplette CD, die in das GuideBuilder-Fenster gezogen wurde, wird mit dieser Funktion ein Amigaguide-File erstellt oder auf Wusch an ein bestehendes Guide angehaengt. Vor der Erstellung ist lediglich auszuwaehlen, ob Textdateien, Verzeichnisse oder Binaerdaten in einem Guide erfasst werden sollen. Bei Texten wird der Pfad zur Text-Datei erfasst und die Anzeige des Textes im Guide ermoeglicht, bei Verzeichnissen und Binaer-Daten werden nur die Pfade zu den Daten im Guide aufgefuehrt. Das I-Tuepfelchen waere hier eine intelligente Erkennung durch den GuideBuilder, welcher Art die zu erfassenden Daten sind und eine entspechende Reaktion bei der Guide-Erstellung. So waeren dann nicht nur die Pfadangaben zu Binaerdaten, sondern auch eine direkte Einbindung/Anzeige von z.B. Bildern denkbar, wie sie bei Texten schon in der vorliegenden Version funktioniert.

Aber auch so kann die AppWindow-Funktion auf elegante Art und Weise fuer die Erfassung von (Pfad-) Katalogen kompletter Verzeichnisse oder deren Inhalt u.ae. genutzt werden.

Ein konkretes Beispiel: Wer alle bisher veroeffentlichten AmigaAktuell- Ausgaben in einem Festplattenverzeichnis gesammelt hat und diese in einem einzigen Guide vereint haben moechte, braucht lediglich den Ordner mit den AmigaAktuell-Guides in den GuideBuilder zu ziehen und das Gadget "Erzeuge" anzuklicken.

Hier zeigt sich dann leider ein Schwachpunkt des Skriptmanagers. Zwar erstellt der Guidebuilder auch sehr grosse Guides und laedt diese in den Editor, ab ca. 1,6 MB Guide-Groesse ist es jedoch nicht mehr moeglich, innerhalb des Editors in diesem Guide zu scrollen oder den Text vor veraendern. Auch eine Vergroesserung der Stack-Vorgabe im Programmicon kann dem nicht abhelfen. Ob ein Programmierfehler oder die verwendete Programmiersprache CanDo die Ursache ist, muss offen bleiben. Wahrscheinlich ist wohl das inzwischen angegraute CanDo der Uebeltaeter. Soll ein solch umfangreiches Guide-File dennoch bearbeitet werden, muesst Ihr zwangslaeufig zu einem externen Editor greifen.

Uebrigens brauchte der Guidebuilder fuer das Verknuepfen des schliesslich ca. 5,7 MB grossen AmigaAktuell-Superguide 7 Minuten und 15 Sekunden.

Neben der Guide-Erstellung aus Text(en) ist oft auch die Integration von Bildern ueber entsprechende Links im Dokument erwuenscht. Fuer den Skriptmanager ist dies kein Problem.

Nachdem ein Guide erzeugt wurde und damit gleichzeitig in den Editor geladen ist, braucht Ihr nur den Cursor auf die gewuenschte Position im Guide plazieren und anschliessend ueber das Menue den Punkt "MakeGuide/Einbinden/Image Extern" oder Tastenkuerzel "rechte_Amiga #" betaetigen. Nachdem im sich oeffnenden Requester die gewuenschte Bilddatei gewaehlt wurde, bringt ein Klick auf "Test" im Guidebuilder das Ergebnis zur prompten Anzeige.

Genauso spielend leicht lassen sich Links setzen, die ausfuehrbare Programme starten: Einfach auf der gewuenschten Cursorposition den Menuepunkt "MakeGuide/Einbinden/Prog.Run" anwaehlen, das zu startende Programm im Requester markieren, fertig. Oder Ihr setzt mit wenigen Mausklicks Links auf externe Guide- bzw. Text-Dateien.

Damit noch nicht genug, als Draufgabe koennen ueber das Guide auf gleiche Art und Weise lha- und lzx-Files eingebunden und nach Anwahl der zugehoerigen Links in die Ram-Disk entpackt werden.

Sowohl fuer die Bilder, die zu startenden Programme, lha- und lzx-Files als auch externen Guide/Texte kann der komplette Pfad ueber die Option "Einbinden/Lokal" beruecksichtigt werden. Aber dann wird das Guide oft auf Grund der individuellen Festplatten-Organisation nur auf der eigenen Rechnerkonfiguration die entsprechenden Bilder/Programme finden. Ist "Einbinden/Lokal" abgewaehlt, wird nur im Guide-Verzeichnis nach den Dateien gesucht.

5.2 ARexx-Unterstuetzung

Neben der eigentlichen Guide-Komposition bietet der Skriptmanager allen ARexx-Programmierern Hilfe und Unterstuetzung an. So koennen direkt aus dem Skriptmanager heraus ARexx-Skripte gestartet werden. Wahleise kann dabei auch die Ablaufverfolgungsfunktion (TRACE) an und abgeschaltet und zusaetzlich jeder einzelnen Programmschritt entweder in einem CLI-Fenster oder in einem eigenen TRACE-Fenster in der interaktiven Programmverfolgung ausgeben werden.

Zusaetzlich lassen sich bei Bedarf alle gegenwaertig geoeffneten ARexx-Ports anzeigen.

Damit aber noch nicht genug, als Sahnehaeubchen ist ein "ARexx Spell" genannte Datenbank zur ARexx-Syntax eingebaut. Befindet sich der Cursor innnerhalb eines in den Skriptmanager geladenen ARexx-Programmes auf einem ARexx-Schluesselwort, so wird bei Anwahl des Menuepunktes "Skripte/ARexx Skript/ARexx Spell" oder ueber das Tastaturkuerzel "rechte_Amiga r" in einem Fenster die Syntax nebst Kurzbeschreibung angezeigt.

In der Amigaguide-Anleitung verspricht der Programmautor fuer zukuenftige Versionen einen weiteren Ausbau dieser ARexx-Datenbank.

Da Heiko Strohmeier zu Recht davon ausgeht, dass nur wenige Amiga-User dieser Funktion wegen ihren gewohnten Editor verlassen werden, hat er vorgesorgt und zwei ARexx-Skript fuer die Anbindung des ARexx-Spellers an externe Editoren beigelegt. So ist es ein Leichtes, ueber den gewohnten Editor die Annehmlichkeiten des ARexx-Datenbank zu nutzen.

Fuer GoldED [3] sind die ARexx-Skripte ohne groessere Aenderung nutzbar und werden lediglich sinnvollerweise in das GoldED-ARexx-Verzeichnis kopiert, die Endung .rexx nach .ged geaendert und koennen nun von GoldED aus direkt ueber "Makros/Makro starten... genutzt werden.

Gleichzeitig kann ueber ein zweites beiliegendes ARexx-Skript ein gerade im externen Editor bearbeitetes ARexx- oder Guide-File an den Skriptmanager uebergeben werden.

Diese Flexibilitaet erhoeht merklich den Programmkomfort und die Akzeptanz, neben dem gewohnten Editor zusaetzlich den Skriptmanager einzusetzen.

5.3 DOS-Skripte

Wurden im Skriptmanager DOS-Skripte erstellt oder geladen, so koenne diese direkt aus dem Programm zu Testzwecken gestartet werden. Laut Anleitung sollen Aenderungen am Skript automatisch bei einem erneuten Start beruecksichtigt werden. Dies klappte bei meinen Tests nicht, es musste immer erst das geaenderte Skript gespeichert werden, bevor die Aenderungen vom Programm bemerkt wurden.

5.4 Guide2HTML

Wer seine Guides ins plattformuebergreifende HTML-Format konvertieren moechte, kann auch dies mit dem SkriptManager auf sehr einfache und bequeme Weise bewerkstelligen.

Dabei hat der Anwender die Moeglichkeit, entweder alle Seiten eines Guides als Einzeldateien oder das Guide als ein einziges HTML-File abzuspeichern. Dabei kann auf das Textlayout in Grenzen Einfluss genommen werden. Um ein Guide nach HTML zu konvertieren, wird selbiges in den Skriptmanager geladen, ueber den Menue-Punkt "MakeGuide/Guide Auslesen" dem Skriptmanager vollstaendig bekannt gemacht und im sich selbststaendig oeffnenden GuideBuilder ueber den Menue-Eintrag "HTML/Guide2HTML/In einzelne Dateien" bzw "/In eine Datei" konvertiert. Auch eingebundene Bilder sowie Querverweise zu weiteren Guides werden beruecksichtigt. In der Dokumentation des Skriptmanagers wird auf einige Punkte hingewiesen, die fuer eine erfolgreiche Guide-Wandlung nach HTML zu beachten sind.

Die Konvertierungsergebnisse sind durchaus ansehnlich, wenn auch je nach Guide-Vorlage und Anspruch an das Seiten-Design u.U. noch nachtraeglicher Feinschliff notwendig wird.

6. Shareware

Skriptmanager ist Shareware, die Registriergebuehr betraegt 20,- DM. In der nicht registrierten Version sind einige Programmfunktionen eingeschraenkt. So beendet sich der Skriptmanager nach einer Stunde Laufzeit, der ARexx- Speller akzeptiert pro Sitzung nur drei Anfragen und schliesst sein Fenster nach 3 Minuten. Ausserdem ist im GuideBuilder die Speicherfunktion gesperrt und die AppWindow-Funktion erlaubt nur einmaliges Drag&Drop.

Zusaetzlich erinnert alle fuenf Minuten ein Nervrequester an die Zahlung der Shareware-Gebuehr.

7. Fazit

Hinter dem Skriptmanager verbirgt sich eine auf den ersten Blick kaum erkennbare Funktionsvielfalt. Das Programm leistet sich nur wenige Schwachpunkte. So z.B., dass DOS-Skript-Aenderungen erst nach dem Speichern vom Programm bemerkt werden und Scrollen und Edieren sehr umfangreicher Texte/Guide im Editor nicht moeglich ist.

Ebenso fehlt eine optische Rueckmeldung im Skriptmanager ueber markierte Buchstaben, Woerter oder Bloecke.

Diese kleinen Fehler schraenken den Gebrauchswert je nach Zielstellung meist nicht wesentlich ein.

Schwerer wiegt da schon die zum Teil zu spartanische oder an anderen Stellen wieder unnoetig verkomplizierte Programm-Dokumentation. Sie erschwert unnoetig den Einstieg in die Funktionsweise des Skriptmanagers und sollte unbedingt ueberarbeitet werden.

Sehr hilfreich waere dabei ein vollstaendiges Tutorial, dass die Erstellung eines Guides, auch mit Querverweisen, von Anfang bis Ende in nachvollziehbaren Schritten darstellt.

Fuer alle Amigianer, die viele Skripte jonglieren, kann der Skriptmanager eine echte Arbeitserleichterung sein.

Die Shareware-Gebuehr erscheint mir fuer den gelegentlichen Nutzer relativ hoch. Bei intensivem Gebrauch und damit auch Nutzen sind die 20,- DM jedoch angemessen.

Ich kann jedem Skript-Freund einen Test des Skriptmanagers empfehlen.
