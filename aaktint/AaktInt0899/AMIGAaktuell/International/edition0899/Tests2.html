Requirements:

Hardware:
  • Kickstart 2.04 (Kickstart 3.0+ for improved presentation recommended.)

  • PowerPC processors are supported
Software:
  • none

Size resp. components of the software:

  • Support of various external devices like scanners/printers

  • Support of any common output device (graphics board/AGA/ECS/Window)

  • Many operators for manipulating of graphics

  • Support of many file formats

  • Intense support of the routines for displaying graphics of the operating system

  • Support of Datatypes

  • Writes graphics as C-source code

  • Can be used as Commodity

  • Integrated screen grabbing tool (up to 24bit)

  • Graphic user interface (can be switched off)

  • Very good locale support

  • Some utilities (picture viewer)
Installation:

First, the 9 archives should be unarchived into a directory. A very good script for the Commodore installer installs the complete SViewII package (or only parts of the package) on hard disk. The section "Installation" of the Amiga guide manual clearly describes, how the optiaml configuration can be found, where to find which library etc.

Overview:

With SViewII the user gets a package, which allows to view, to convert and to edit graphic files of various formats.


SViewII is extremely modular, has no problems with comressed files (XPK or PowerPacker) and also supports the present Datatypes.

    Practical test:

    The first impression is not very good (at least with the test configuration; see above), because SViewII needs very long for the startup, because any present operator and module is loaded. There are start icons for the deired configuration (for AGA or CyberGraphX), but they don't speed up the start process very much.

    After that the graphical user interface opens (if you want).

    Using the interface you can control any operation:

    • Using "Medium" the input/output device can be selected (that is HD, Clipboard or a mounted device).

    • Using this medium you can load/save, view or edit files

    • "Screenmode" allows to set the screen size for the viewing of the pictures, if a screen has been set for output.

    • There are four index card objects, driver, operators and screens. Using these index cards you can control any function.

      For the contents of the index cards there are four gadgets, which can be used for changing the contents of the cards.

      Using "SVInfo" you get informations about the import filters. Using "?" it is possible to set the methods how the filter handles the file.

      Using "Add" new filters, drivers or operators can be added. Using "Remove" filters, drivers or operators can be removed.

    Objects:
    This index cards contains any format SViewII can process. If you want to convert a picture, it has to be loaded, the save picture type to be selected and the picture to be saved.

    Drivers:

    This index card contains any available output device (like screen drivers for Picasso96, AGA, ECS, CyberGraphX or drivers for output in a window).

    Operators:

    This includes any editing function, which allow editing of the pictures.

    Screens:

    This lists any open screen. If needed, the contents of a screen can be saved.


    But not the interface alone allows using SViewII. SViewII offers an ARexx interface, which allows to control many actions (like e.g. converting of whole file lists) automatically.

    Or you can control SViewII using the CLI.

    So a general controlability is guaranteed, so anyone can access the possibilities, SViewII offers.

    From time to time additional operators and drivers will be added. So there is a driver available, which allows to import GL based data (OpenGL data to AmigaMesaRTL).

    Positive:
    • extensive manual as AmigaGuide file

    • very good help system

    • many, many import- and export filters

    • many operators for picture editing

    • Modularity/expandability
    Negative:
    • Sometimes hesitant in speed

    • Interface limited to the functions
    Conclusion:

    Who already knows the various Datatypes of Andreas Kleinert, knows that the author has a knowledge of graphics on the Amiga. SViewII unifies any possibilities of the Datatypes of A. Kleinert and offers in addition very much options of modification. The flexible handling of SViewII either using the graphical user interface or the CLI is quite welcome.
