Test reports

2. Report: Amiga-Emulator-package >>AmigaForever<< - Part 2 (by K.-H. Bader)

I used WinUAE Version 0.8.8 R8 so far. The newest version is 0816release4. It emulates a 68040-CPU. That means: learning by doing. I created a new drawer on my harddisc with the newest version inside. Beside that the newest AmigaOS, that means AOS3.9 with BoingBag1, which corresponds to Workbench-Version 45.2

As a screen for testing I use "UAEgfx 800x600 8-bit 60 Hz". OCS, ECS or AGA can be choosen. At first I used OCS. To boot to Workbench the system needed ca. 40sec seconds, the time could be reduced by give up the background picture and loading any programs. With ECS it lasts around 30 secs and on AGA 35 secs.

The actual WinUAE-Version 0816 R 4 is usable for me again.

With other versions e.x. 0816R1 the clock ran to fast and typing in a shell was impossible because of timing problems.

Now I use 68040-CPU-8 MB Chip - 64 MB ZorroIII Fast.

It is a real pleasure loading any pictures with Multiview, that runs really faster than on my Amiga1200 030/40.

As I emulte a Picasso96 graphics card I use the ECS-chipset. Reffering to the docs BSD-socket-library does not works correctly until now, so I start testing with YAM, if there are any errors. I cannot assert any problems yet.

I noticed that the actual version of WinUAE does not mix all the cards of my SKAT-game correctly, I get always the same cards. Well, I can do without the game, as the computer is not very hard to beat, IMHO.

I have still to test the following things:

The manual says that the name will me taken over, but yet I just got "WinCD_H:" and have to set a assign to use some parts of a CD.

Well, that's enough for today. If someone wants to give hints, just email ! Even questions will be answered happily - as far as I can help. Up to the next part !
