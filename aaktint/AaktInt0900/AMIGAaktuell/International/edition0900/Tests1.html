GradientSlider/ColorWheel

Some of you might already know that Georg Steger is working on the operating system AROS, which is trying to implement an operating system on Wintel PCs that is comparable and source-compatible to the AmigaOS. But Georg Steger isn't just planning a simple port of the AmigaOS, he also wants to implement improvements (which he hopes will be downward compatible). Two of these improvements were recently ported back to the original OS by Stephan Rupprecht. We are talking about the system classes ColorWheel and GradientSlider.

Installation does not require the Installer from Amiga and is very simple. Just get the two archives from Aminet (or from the sources below), unpack them and copy the two binaries gradientslider.gadget and colorwheel.gadget into the classes/gadgets directory of your boot partition (usually the Workbench partition). That is it. There is nothing to configure.

Programs like "IconEdit" or the fractal renderer ChaosPro now use these classes just like the original classes that came with the AmigaOS. But Hi- and True-Color screens now get a colorful palette. And these classes are also recommended for AGA/ECS users (just take a look at the screenshot and you will see what I mean...)

Displaying the colorwheel takes a little longer, but the result is worth the small performance loss.

DeliTrackerII V2.32

Well, I don't really think that I have to introduce DeliTrackerII anymore, it's even found on every Aminet-CD. But, what the hell? DeliTrackerII is THE music player for the Amiga. No matter if you use Sound/Fast/Scream/ProTracker modules or more exotic formats like NSF, PumaTracker or XTracker, DeliTracker plays just all those without problems. Even if you come across a module that spits the dummy, you can find additional decoders on aminet that can be added easily to DeliTracker. The user interface looks rather old-fashioned (Hello, AmigaOS2.0! ;) ), but don't judge DeliTracker's capabilities by its looks.

Currently I can't think of anything useful for a module player that DeliTracker doesn't feature.

Be it ARexx- or XPK-support, fast high quality players and easy handling - DeliTracker has everything. DeliTracker is very modular - for each module format, you need a suitable decoder-module (a so-called player). The Aminet distribution contains about 100 different players.

There is also a player for DataTypes, so sound files like AIFF, WAVE or even MP3 are no problem. Sound output is also handled by modules (called Genies), but these Genies can do more than just play sound. You can also convert files to other formats, open spectrometers or display information about the files you are playing.

Es gibt sogar ein HappyNewYear-Genie und ein Weihnachts-Genie, das Glueckwuensche fuer den Jahreswechsel bzw. Weihnachten ausgibt (und sonst nix weiter ;) ).

There is even a HappyNewYearGenie and a Christmas-Genie, which display good wishes for Christmas or the New Year (and nothing else ;) ). The Player for DigiBoosterPro from Aminet opens up the possibility of writing your own Audio-CDs (for your private use only!), because it supports the sound system AHI. This way, you can easily convert DB-Pro modules to AIFF or WAVE and burn them onto CD with MakeCD or BurnIt.

The bad news: development for the Amiga stopped after this version. The good news: DeliTrackerII is now freeware. You don't have to pay for a licence anymore. But since DeliTracker is very modular, skilled programmers shouldn't stop writing new Players and Genies. This could circumvent the lack of development for the most part. If you have a Windows computer, you can get the shareware version for Wintel computer (and of course register it, if you like it).

System requirements cannot be easily defined, because some Players or Genies use more or less RAM or CPU power. For simple Protracker or MED modules, a "stock" A600 should suffice. For DigiBoosterPro or any other modules with more than 4 channels, the requirements are a little higher. The absolute minimum would be an Amiga with 150KB of free RAM and at least OS2.0.

Author: Frank Riffel, Peter Kunath
Aminet: mus/play/DeliTracker232.lha
Program version: 2.32
Homepage: http://www.deliplayer.com (Windows-Version)
Licence: Freeware

Qt 2.0

Qt 2.0 is a player for Quicktime animations. And, as the name suggests, it is the second version ;). There is no user interface - the arguments are given via shell or ToolTypes. The distribution contains several versions for 68020 to 68040/FPU. These are optimized "by hand" but only compiled with an appropriate compiler flag.

There is no special version for 68060 CPUs. Users of this processor should use the "normal" version. A PPC board is (once again?!) not supported.

Quicktime animations can be played directly from hard disk or CD-ROM. To get reasonably fluid animations, your drive should be able to deliver at least 2 to 3 MB of data per second. A 32x SCSI CD-ROM drive (or a fast drive with DMA capability) should do the trick. For smaller animations, a 12x SCSI drive should suffice. Compressed QT animations can also be read from a slower drive, but in this case the processor's name should contain at least two sixes (68060!). When enough RAM is available, you can also load the whole animation into memory. A graphics board would be helpful, too.

Unlike other players for the Amiga, Qt2.0 is also able to save the video or audio stream as ILBM files and 8SVX sound. These files can be easily converted by other programs (Wildfire, ADPRo, CDGXL). Qt also has the usual features like Floyd-Steinberg/Ordered/Gray dithering, CyberGFX, HAM and public screens as well as sound output via AHI. For CD32 users, the WriteChunkyPixel mode (WCP) might be interesting, because this makes (as far as I know) an accelerated output possible by using the Akiko chip (but I'm not entirely sure!!).

Qt 2.0 supports these quicktime formats:

Video-Codecs:
-RLE
-CinePak
-Road Pizza
-JPEG
-SMC
-RAW
Audio-Codecs:
-MACE
-IMA4

The "Intel Indeo" format is not supported due to high licence fees. The Qt archive also contains the program "Flatten". I will have to digress a little to explain why you need this program. Quicktime was invented by Apple. As you know, Apple produces the Macintosh computer, which runs MacOS. The filesystem of MacOS is called HFS (HFS+ for MacOS8 and higher; HFS stands for "hierarchical filesystem", okay, I'm digressing a lot already ;) ). In the HFS filesystem, files are split in two parts. The first part is the so-called "resource fork". You could compare it to the ".info" file on the Amiga, but it contains more information. The second part is the "data fork". This part contains the data itself. And since Quicktime has been around for a while on Macintoshes, there are a lot of animations on Macintosh CDs (or other HFS formatted media). So if you are reading Macintosh CDs on an Amiga (CacheCDFS from AmigaOS3.5 is good for that purpose), the files are displayed like this:

datei1.rsrc
datei1
datei2.rsrc
datei2
...
etc.

The actual output may differ, depending on your CD filesystem. Qt 2.0 can't handle those split files, and that's where "Flatten" comes in. After you've started Flatten, you can drag the data fork onto an AppIcon. Flatten asks for the target file and then combines the resources fork with the data fork so that Qt is able to show the animation.

I hope this digression has made Flatten more understandable ;).

Now a few tips for Amiga users without a graphics board:

1) Get the patches FBlitV3.56 (http://www.tpec.u-net.com) and BlazeWCP (Aminet: util/boot/BlazeWCP.lha) and install them as described in their manuals.

2) Play the Quicktime animations in Grayscale mode to make the playback less jerky.

3) If you have installed BlazeWCP, you should start Qt 2.0 with the "WCP" option. This makes it use the faster chunky2planar routine from BlazeWCP. Play around with the settings a little.

4) With the parameter "EVERY", you can define "Frame-Skips". For example, "EVERY=2" displays only every second picture of the animation. This may help if the sound is played faster than the pictures.

Image2Icon:

And last but not least, Image2Icon. I have no idea how to describe this program - the name says most of it. Image2Icon (short "I2I") takes a picture file and creates an icon with a smaller version of the contents. Or shorter: I2I creates a thumbnail icon from a picture file. I2I isn't the first program to do this, as some of you surely know, but I think it creates the best visual quality with AmigaOS3.5. I2I offers very fast and good dithering algorithms and takes the aspect-ratio of the file and Workbench into account - the thumbnail of a PAL:HighRes (640x256) picture doesn't look too distorted on a Workbench with 800x600 pixels. Even on a screen with only 16 or 32 colours you can make out what the picture shows thanks to the great dithering.

On OS3.0 and OS3.1 systems, the pictures are saved in NewIcon format, under OS3.5 the Color-Icon format is used. You can use any picture formats you have a DataType for. If you want more information, you can have I2I write the dimensions inside the icon, too. You can even define the position and the font to be used.

If you start I2I from Workbench, it creates an AppIcon you can drop pictures on. Alternatively, I2I is useable from a shell window. You will need at least OS3.0. The manual states that HAM graphics are not supported, but oddly it worked for me without any problems.
