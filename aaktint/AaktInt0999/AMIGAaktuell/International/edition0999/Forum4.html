4. Comment concerning the Linux Kernel Decision of Amiga (by Andreas Weiner)

»According to my oppinion the change from QNX to Linux as the kernel for
the AmigaNG was entirely right.

For this there are lots of reasons:

1. Linux is completely stable.
2. The Linux kernel easyly fits on a disk (it is relatively small)
3. The Linux kernel can be adapted (size and abilities of the kernel
are flexible)
4. Linux already has a good software basis growing steadily.
5. A major part of Linux software is Open Source Software.
6. Linux is backed by a great number of people with idealism and
know-how.
7. The Linux developement has breathtaking speed.
8. Especially in networking Linux is very able (concerning speed
and applications).
9. If really the Transmeta would be choosen as new CPU for the AmigaNG
the kernel was already adapted/optimized for it ...

Even if speed and size might speak in favour of QNX it misses one
decisive element: The basis of software (which certainly Amiga will
need for survival) and a big community backing it up.

Possibly the steadily growing Linux community can as well be "converted"
to Amiga, what might be quite advantageous for Amiga.

Now we'll have to wait for it, how quickly (and well) Amiga will execute
their piece of work for AmigaNG, I am eager for the result.

However the tendency of the whole is absolutely correct.

Andreas Weiner «
