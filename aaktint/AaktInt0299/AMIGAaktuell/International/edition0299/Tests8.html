8. Software test: mathematical program PicFX (by Axel Teichmann)

»   Tested program: gfx/edit/PicFX  Revision: 0.85 Alpha  Type: freeware  Author: maxime@intelcom.ch (Maxime Gamboni)  Tester: A.Teichmann@gmx.net  Test duration: 1 day  Used hardware: Amiga4000/040 OS3.0 25MHz Cybervision64/3D CGX  System    16 MB Fast RAM / 2 MB Chip RAM     Installation:

PicFx has no installer script or something similar, it can copied completely anywhere you like as you unarchived it and start it then.

Design:

The program uses MUI and set shortly after the start a 24 bit screen, which can be altered, of course.

Practical test:

PicFX is not stable enough. Some crashes happen to often. With PicFX you can mix pictures using a mathematical formula. Unfortunately this doesn't work without intensive reading of the manual to understand how such inputs have to be done.

After creation of a project (0) with a picture (0) and creating of another project (picture 1/ project 1) the program can mix these data (0/ 1) with another (seperately for R, G, B). In any case the output is done into another project, which has to be opened before. This makes it possible to combine nearly any pixel of project 0 with any pixel of project 1. There are more than one project possible, so there are infinite possibilities.

The formulae are freely selectable, but to enter cryptical. There is no preview function, which could show the result before, so you have to "hope" that the result is as you wanted it to be.

PicFx is no "picture shearer" à la Wildfire\7, but it's more suggestive to the function of Animage & Maple V, just limited to pictures.

Positive:

Good, useable interface, funny/ interesting results.

Negative:

Hasn't reached the beta stadium yet, crashes to often. No preview function, cryptic inputs.

Conclusion:

PicFX is interesting for people, who like playing with mathematic formulae, or want to combine pictures in an interesting way. The program needs a long settling-in period, to understand what is done. This program has no value for users who want to quickly shear or mix pictures of some sort.

Axel Teichmann <A.Teichmann@t-online.de>«
