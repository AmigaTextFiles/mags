1. Infos about the 24bit graphics board BVisionPPC (by Steffen Haeuser)

»The BV/PPC is not an Autoconfig board as well as the CV/PPC. If you want to use WarpUP programs, that open own screens you have to set these ENV variables, to configure the board manually:

(Just type in a Shell. After typing once it is stored on hard disk and is from then always active): 

echo >env:powerpc/force "1"
echo >envarc:powerpc/force "1"
echo >env:powerpc/gfxaddr "$e0000000"
echo >envarc:powerpc/gfxaddr "$e0000000"

A reboot is recommended.

By the way: Due to the "Anti-WarpUP-Protection" in CGX PPC it cannot be used with WarpUP or rtgmaster - use CGX 68k instead. This makes no difference because CGX PPC does not have any advantages in performance, it's a kind of "Software-Dongle".

To make rtgmaster run with the board there should be WarpUP 3.1 installed (powerpc.library V14.6). If there should occur prorblems with ppc.library's "Anti-WarpUP-Protection":

  • Don't install CGX PPC, ppc.library based datatypes, RC5-ppc.library, PPCInstall, or ElfLoadSeg.
  • With an A1200 PPC. One of the newer ppc.library (approx. 46.19) set env:powerpc/TERMINATOR and envarc:powerpc/TERMINATOR to "2", as described above, an "older" version (approx. V45) to "1". This removes the "Anti-WarpUP code" (the remover is only being activated, if a WarpUP or rtgmaster program starts. If this is not the case the ppc.library programs continue running.
  • In the moment of the start of a WarpUP or rtgmaster program there should NOT be any ppc.library programs active in the background.

This type of installation is also important with the Warp3D PPC version.

Steffen Haeuser«
