7. Software test: eMail-program Iris (by Axel Teichmann)

»   Tested program: comm/mail/IRIS  Revision: 1.7  Type: shareware (10 DEM, approx. 5.10 Euro)  Author: M.Tjoelker@nl.cis.philips.com (Jilles Tjoelker)  Tester: A.Teichmann@gmx.net  Test duration: 3 days  Used hardware: Amiga4000/040 OS3.0 25MHz   16 MB Fast RAM / 2 MB Chip RAM  Used software: Miami V2.0 / T-Online Account     Installation:

Iris is being installed using an installer script, which is clearly programmed and has no problems with eventually hard installation environments.

Design:

In opposition to other mail programmes (e.g. FFnews/YAM/MD-II) Iris refrains from MUI. The complete interface is realized usign BGUI.

Due to this fact the interface is not as perfect in colour and style as e.g. other mailers and it gives the impression of drearyness. But Iris offers other qualities, you might only know from YAM. Iris is shareware and offers at each start/ end a requester, but somenone can live with that and a shareware fee of 10 DEM (approx. 5.10 Euro) is justified.

Pracical test:

After the start you just have to call the menu point Configuration/Config to enter your personal data like eMail address, real name, POP and SMPT servernames and the user name/password. After that Iris is completely useable. After starting a TCP stack you can receive your eMail via a click on a button. Parallels to YAM are not to ignore.8-) Writing the mails is only possible using the standard editor "Ed",what cannot be adjusted seperately (only via an env variable), but someone can even live with that.

Even without MUI everything can be configured, like e.g. the post box, filters, directory, fonts.

Iris offers a powerful ARexx port, which leaves nearly no wishes. It makes it possible to create/delete/send, etc. mails. Iris works without crashes and the multiple running of Iris doesn't result in a Guru, but an additional use for other accounts.

Positive:

Easy to configure, easy to use. &uot;BGui.library" is included. Due to the missing MUI lots of memory are used less than its rivals. The start of more than one Iris application is possible.

Negative:

Interface design without optic qualities/ not modern (my opinion). Filter unfortunately offers the "move" function. External editor can be selected ussing an env variable only.

Conclusion:

This program is made for small Amiga, which don't have enough memory at all to install a complete MUI system with all the MUI classes. And Iris hasn't to hide behind other mail programs, because it offers nearly the same power with small amount of rescources.

Axel Teichmann <A.Teichmann@t-online.de>«
