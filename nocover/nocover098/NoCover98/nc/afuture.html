dynAMIte 2.0 ist erschienen 1.5.2002
2:00
dynAMIte ist in der Version 2.0 erschienen.

Hier die Original-Meldung:

The latest achievement in pyrotechnics: dynAMIte 2.0 is now available for free download!

Back from a fantastic trip to hamburg, where we demoed our latest #amigazeuxAnywear collection, we wanted to take the time to get the Amiga community caught up on our flagship project, dynAMIte.

Contrary to the nonsense that is being spread about legality of other products, dynAMIte is more than alive and well and we have very exciting plans for the future with amazing new products introducing several other exciting and amazing Amiga-like, mind-blowing features.

#amigazeux have assembled the cream of map artists to work on something completely different, guaranteeing both high quality and high performance in the final product and ensuring that dynAMIte 2.0 and beyond will be a true Amiga product, created with due care and attention to the philosophy and practices that have always made Amiga solutions the envy of the world.

We and probably several others are wowed and amazed and excited to announce the availability of version 2.0 of our exciting product dynAMIte.

Changes:

  • Artificial intelligence made by Illka "iti" Lethoranta has received a complete overhaul resulting in AI2.0
  • Client will now close the connection to the server if it hasn't received a ping for 30 seconds.
  • Added a check for mapjump pickup to prevent maps being played over and over again.
  • Fixed a bug when deallocating bubbles after they have been used.
  • Bots running on the server are now able to do bubbles too.
  • GSL now displays number of bots.
  • Added a workaround for amarquee v53 session ip bug. Now gets the hostname of the session, could break clone detection in some rare cases.
  • Fixed a serious bug which caused the game to never end when someone died exactly when hurryup mode was activated.
  • Slightly changed Teleport-All powerup. Instead of teleporting every player to a random position, players will now be teleported to the position of the player who picked up this item.
  • Fixed a bug where kick/remotebombs caused invisible blocks.
  • Bots are now able to kill themselves.
  • Keymappings are saved now.
  • DServer will now stop a game if no human player is participating in it.
  • Initial bombs will no longer turn into blocks if a player picks up a bombs2block bonus. This was changed back to old way because several maps are created to use this behaviour and didn't work that well with the new behaviour.
  • DServer never closed Amarquee sessions since 1.9. Fixed.
  • Added new powerup for now missing flashbombs -> darkness.
  • Flashbombs have been removed because of lack of actual usefulness.
  • If DServer was listed in GSL and it was quit, it was very possible httptask simply stalled because of synchronous message communication with DServer (DServer didn't check msgport while waiting for httptask to quit)
  • You could drop as many flashbombs as you wanted. Fixed. Obsolete bugfix, see above.
  • It's now possible to set maximum number of CPU players in DServer.
  • Increased treshhold of flood detection a bit to make it possible to kick 8 bots in a row :)
  • DServer now displays the local ip in the status window on startup.
  • Recompiled again with AmigaE 3.3a because of some unexpected problems.
