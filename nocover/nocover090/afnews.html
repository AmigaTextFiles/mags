Hier die Originalmeldung:

PrintManager v39 is a so called printer-spooler, it intercepts all printer. device calls to the parallel-/serial.device and saves the printer specific raw data to a file. This way, programs sending data to the printer don't have to wait for the slow printer to continue. PrintManager also gives you some control over the data send to the printer.   • caches only printer.device calls
  • can cache all calls to parallel-/serial.devices
  • allows to exclude certain tasks from spooling
  • nice gui which works with proportional fonts
  • files can be printed from inside Printmanager (uses datatypes.lib)
  • works with datatypes.library v45 (v45 recommend, not required!)
  • own FAST parallel.device (registered version only)
  • localized
  • after reset/crash unfinished printings are recovered
  • delayed printing
  • print count function
  • allows to redirect printing to another device
  • upto 10 devices can be spooled
  • ...
