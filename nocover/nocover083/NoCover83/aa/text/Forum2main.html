2. Leserbrief zu »Abschließend« in Ausgabe 01/2001 (von Steffen Häuser)

Ich kann zu Martin Bautes Schlüssen in "Abschließend" in keinster Weise 
zustimmen, dass "OS 4.0 PPC" eine "bescheuerte Idee" wäre.

AmigaDE ist leider auf lange Sicht (= voraussichtlich Ende 2001) nicht 
verfügbar. Software-Firmen brauchen etwas, wofür sie Software verkaufen 
können. Eine Erklärung "Wir machen OS4.0 PPC" wäre ein Signal für den Markt 
"Es geht noch was auf dem Classic, ihr könnt ruhig noch Classic-Software 
kaufen" und damit ein direkter Support von im Classic-Markt aktiven Firmen. 
Letzten Endes würde eine Stabilisierung des Marktes auch Amiga selbst 
wieder zugute kommen, für den AmigaDE (u.a. auch durch Erhöhung der 
Verkaufszahlen neuer Hardware, die sowohl für den Classic-Amiga als auch 
den DE verwendbar ist, ebenso würde IMHO Amiga davon profitieren, wenn noch 
eine "aktive Community" vorhanden ist, und hierauf würde sich eine 
Ankündigung von OS 4.0 PPC sicher positiv auswirken).

Ein OS 4.0 PPC könnte ich mir etwa so vorstellen:

- 68k-Emulator (der ja im Prinzip schon fertig ist)
- PPC-native-Versionen von exec, dos, graphics, intuition
- Neues Filesystem, das beim Seek nicht mehr so lahm ist (für 68k- und PPC-
  Systeme)
- PPC-native-Filesystem für zumindest die wichtigsten Devices (= CS PPC UW
  SCSI, Blizzard PPC SCSI, A1200/A4000 IDE)
- Neues Kickstart-ROM (für 68k- und PPC-Systeme)
- Eine WB-Emulation, die vollständig ins System integriert ist, voll PPC-   
  native
- Ein komplett PPC-nativer Sound-Standard, voll ins System integriert
- Erweiterung von WarpOS: Einbindung von virtuellem Speicher, zumindest 
  applikationsbasiert
- Erweiterte lowlevel.library mit PPC-native-Funktionen
- PCI-Treiber für die wichtigsten Dinge im OS enthalten

Eine Erweiterung auf einen Emulations-Layer des Classic-OS auf AmigaDE kann 
man ja später immer noch machen, ist meiner Meinung nach auf lange Sicht 
sogar sinnvoll, aber die PPC-OS-Portierung wäre der erste Schritt. 

Es muß ja nicht eine komplette Portierung des OS sein. Wie man an diesem 
"Vorschlag" sieht, müsste "OS 4.0" nicht nur für PPC sein. Gerade die 
Verbesserung des Filesystems und ein neues Kickstart-ROM (und Integration 
von Treibern für aufkommende PCI-Lösungen) würden sicher auch 68k-Käufer 
ansprechen.

Kurzum: Ich halte ein OS 4.0 für eine sehr gute Idee.
