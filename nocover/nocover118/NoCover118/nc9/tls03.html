MagicCX

DAS ultimative Commodity für den Amiga mit OS 2.0 und höher

*** MagicCX ist ein SASG Produkt ***

- Shareware -

MagicCx ist ein System-Commodity, für alle Einstellungen die sonst nur mehrere Tools-Programme ausführen können. Mit diesem Commodity läßt sich das Amiga-System komfortabel einstellen. Folgende Einstellungen können vorgenommen und nur nur in der registrierten Version gespeichert werden:

  • Die Alarm Einrichtung (extern).
  • Spezielle Analoguhr (extern).
  • Das interne Blanker System.
  • Ein spezieller Kalender (extern).
  • Die Titelzeile/Fenster Uhr.
  • Verwandle Cycle Gadgets in PopUp Menüs (extern).
  • Explodiere/Implodiere Fenster (extern).
  • Disks beobachten mit Autoformatier Einrichtung (extern).
  • Definieren Sie Ihre eigenen Funktionstasten (extern).
  • Greife/Drucke aktive Screen/Fenster (extern).
  • MagicCX`s Hotkeys definieren.
  • Fenster jagen.
  • Tasten Fenster/Screen Blättern.
  • Fenster vor MagicCX`s Fensteraktivierung bewahren (extern).
  • Verschiedenes-1 (kein Drive-Click...).
  • Verschiedenes-1 (Tasten-Click...).
  • Maus Fenster/Screen Blättern.
  • Mausbezogene Einrichtungen.
  • Vergrößern/Bewegen von Fenstern mit einem Klick (extern).
  • Selektiere ein(en) Screen/Fenster (extern).
  • Spezielles Fenster Blättern.
  • Forciere alte Screens auf neue (extern).
  • Kalkuliere Telefonkosten (extern).
  • Substituiere Default Projekte (extern).
  • Fensterbezogene Einrichtungen.
  • Der ARexx Port (inklusive RCS Modul) (extern).
  • Bessere "Please insert..." Requester (extern).
  • Nutzen Sie ".."/"." auf Ihrem System (extern).

MagicCX ist voll OS 2.0 kompatibel. Dadurch natürlich auch mit dem entsprechenden "Look and Feel". Voll lokalisiert unter OS2.1!

MagicCX installiert sich als Commodity, so daß Sie also all die bequemen Dinge nutzen können, die ein Commodity bietet.

Komplett neues Konzept. Komplexe Modul-Eigenschaften. MagicCX ist ausbaufähig durch die Verwendung von Modulen. Sie können MagicCX so konfigurieren, wie Sie es möchten. Nur die benutzten Eigenschaften werden geladen.

Benötigte/Benutzte Module werde in einem Verzeichnis "Modules" installiert. Dies entspricht zuweilen dem Konzept der Workbench, wie sie es für Treiber nutzt.

Modulares Blanker Konzept, mit zwei internen Blanker Typen. Die Blanker sind natürlich ausbaufähig.

Für Benutzer von OS 3.0 oder besser existiert ein Modules Manager, mit dem Sie Module installieren/deinstallieren können. Es ist nicht mehr nötig die Workbench zu nutzen, um Module herumzuschieben.

MagicCX ist aufgeteilt in viele verschiedene Teile. Das Server Programm, das als Hauptschaltzentrale dient und installierte Module nutzt. Der Preferences Editor, mit dem Sie MagicCX konfigurieren.

MagicCX nutzt eine hervorragende Gadget Layout Engine, um Ihnen das Beste an Fenster Layout bieten zu können. Volle font-sensitive und shortcut-unterstützende Fenster sind das Ergebnis.

MagicCX hat viele Module, aus denen die User wählen kann, um Ihnen einfach das Beste zu geben

Die Installation mit dem Commodore-Installer ist leicht und man braucht nur das entsprechende Installation-Icon anklicken, wobei man unter einem deutschen und einem englischen Install-Icon wählen kann.

MagicCX`s Hauptprogramm kennt einige ToolTypes, die entweder innerhalb der ".info" Datei verzeichnet sein sollten, oder aber beim Aufruf aus einer Shell benutzt werden können. Falls nicht anders gewünscht, so wird nach dem Start von MagicCX ein AppIcon auf der Workbench dargestellt.

Das Hauptziel von MagicCX liegt darin, dem User das Beste an Flexibilität und Konfigurierbarkeit zu geben.

Flexibilität wird erreicht durch einen völlig neuen Modulmechanismus, eingeführt in die Welt der Commodities durch MagicCX. Diese Module erlauben es dem User Funktionen zu wählen, die durch MagicCX benutzt werden sollen, um Ihm die Möglichkeit zu geben ein Commodity zu gestalten, das seinen Wünschen entspricht.

Konfigurierbarkeit wird erreicht durch ein extrem benutzerfreundliches Preferences Programm, das keines der modernen Eigenschaften eines fensterbetriebenen Programms vermissen läßt. Es ist Font-Sensitiv in jedem Fenster. Alle Gadgets sind mit Shortcuts versehen. Das klare und offene Design aller Fenster gibt dem User alles was er braucht, um das Programm einfach zu konfigurieren.

Leider hat man bei einer unregistrierten Version nicht die Möglichkeit alle konfigurierten Einstellungen zu speichern.

Eine deutsche Anleitung liegt dem Programm im "AmigaGuide-Format" bei.

Die Minimal-Konfiguration für MagicCX besteht in einem beliebigen AMIGA-Computer mit mindestens AMIGA-OS 2.0 und 1 MB RAM. Da die Module komplett im RAM gehalten werden, gilt hier: Je mehr je besser.

Eine Festplatte ist nicht zwingend notwendig, aber sehr ratsam.

Fazit:
MagicCX ist wieder mal eine gelungene Produktion der SASG und jeder Amiga-User hat wohl seine Feude an der registrierten Version daran. Wenn dann noch alle schnelleren Prozessoren unterstützt und einige neue Screenblanker implentiert werden, ist MagicCX ein Tool was jedem zu empfehlen ist.
