RunLame & RunLame Infiltrator

Wer kennt es nicht das Problem mit den alten Spielen und Programmen die nicht System Konform Programmiert sind, entweder muß man die Cache`s ausstellen oder in den Pal Modus zurück schalten (wie Ich) oder iegendwelche anderen wiederigen umstände verhindern den Programmstart. Es gibt da zwar eine menge von Degradern die diesen Job erledigen aber entweder muß ein Script geschrieben werden oder der Degrader muß zuerst gestartet werden was mir auf die dauer echt auf die Nerven ging. Aber als Ich meine CD`s durchsuchte stieß ich auf einen Degrader den man vor das Programm Linken kann was Ich für sehr nützlich halte, weil einmal an das Programm gelinkt und nie wieder einen Degrader gestartet.

Was bittet dieser Degrader:
  • Ausschalten von den Prozessor Caches
  • Man kann das Vector Base Register (VBR) ins Fast Ram Verschieben
  • Der Screen Display kann auf OCS oder ECS gesetzt werden
  • Die Sprites können auf OCS/ECS Lowres zurück gesetzt werden
  • Die Sprites können auf eine Systenkonforme weise entfernt werden
  • Flexibeles Fixing für viele schlecht geschriebene Programme
  • Läuft unter Kickstart 2.04, 3.0 und 3.1
  • In 100% Systemfreundlichem Assembler Code Programmiert
  • Vom CLI und Workbench startbar

Naja von RunLame war Ich ja nicht so begeistert aber vom RunLame Infiltrator, es ist im großen und ganzen das selbe wie RunLame aber man kann ihm an das Programm Linken damit man die ganzen anderen Degrader getrost vergessen kann.

Mein Tip einfach mal ausprobieren.
